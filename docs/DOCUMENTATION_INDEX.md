## Contributing to Documentation

### Reporting Issues

If you find errors or unclear sections:
1. Create GitHub issue
2. Specify document and section
3. Describe the problem
4. Suggest improvement if possible

### Suggesting Improvements

To suggest documentation improvements:
1. Create GitHub issue with "documentation" label
2. Explain what's missing or unclear
3. Provide use case or example
4. Offer to help write if willing

### Writing Documentation

When contributing documentation:
- Follow existing style and format
- Use clear, concise language
- Include examples
- Test all instructions
- Proofread carefully
